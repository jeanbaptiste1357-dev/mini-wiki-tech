Le workflow Git

Le workflow Git basé sur les feature branches commence sur la branche principale, généralement appelée main. Lorsqu'un développeur veut ajouter une fonctionnalité ou corriger un bug, il crée une branche dédiée. Il y réalise ses modifications et ses commits sans toucher au code principal.

Une fois le travail terminé, il pousse sa branche vers GitHub et ouvre une pull request. Cette pull request permet de présenter les changements et de préparer leur intégration. Après validation, la branche est fusionnée (merge) dans la branche main, et la branche de développement peut ensuite être supprimée.

Les avantages

  1. Séparer clairement le travail en cours du code principal.
  2. Faciliter le travail en parallèle entre plusieurs développeurs.
  3. Permettre de tester une fonctionnalité sans toucher à la branche principale.
  4. Garder un historique propre et organisé.
  5. Rendre les revues de code plus claires via les pull requests.
  6. Détecter plus facilement les conflits ou problèmes avant le merge.
